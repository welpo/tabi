Enabling (and positioning) the Table of Contents

There are two ways to enable the Table of Contents (ToC). If you want it to be right below the header (like in this page) set this variable on the post‚Äôs front matter:

[extra]
toc = true

If you‚Äôd rather show the ToC elsewhere on your post (e.g. after an introduction), you can do so by adding a line with this content wherever you‚Äôd like the ToC to appear:

<!-- toc -->

You can also use the simple {{ toc() }} shortcode, which will simply write that string for you, effectively inserting the ToC (Michael Clayton‚Äôs idea).

This method will render the ToC without the ‚ÄúTable of Contents‚Äù header. This allows you to use a different (or no) header for the ToC, or hide it like this:

Setting a maximum depth

You can set the maximum depth for the ToC by specifying the toc_levels variable, which takes an integer between 1 and 4:

[extra]
toc_levels = 2

In this example, only the first two levels of headers would be included in the ToC, regardless of their actual HTML tags (h1, h2, h3, etc.). If you want to include only the main level of headers, set toc_levels = 1. The default toc_levels value is 3.

Keep your readers in mind when setting the toc_levels. While it can be tempting to include many nested levels for detailed navigation, a shorter and simpler ToC can often be more reader-friendly and less overwhelming. Adjust the depth according to the complexity and length of your content for the best reader experience.

Hiding headers from the ToC

You might want to hide certain headers. For example, if your article has many Figures or Tables, they might clutter the ToC. You can hide specific headers in the ToC with the toc_ignore_pattern variable.

This variable expects a regular expression (regex), as it‚Äôs using Tera‚Äôs matching test. The toc_ignore_pattern is tested against the text of the header, excluding the # character(s). For example, for the header ### Further reading, the text Further reading would be checked against.

Here are some example values for toc_ignore_pattern along with the headers they can hide:

toc_ignore_patternExcludes headers which‚Ä¶
Tablecontain ‚ÄúTable‚Äù
^Figurestart with ‚ÄúFigure‚Äù
(?i)(table|figure)start with ‚ÄúTable‚Äù or ‚ÄúFigure‚Äù (case insensitive)
\[Draft\]$end with ‚Äú[Draft]‚Äù.

You can test your regular expression on a site like regex101 to ensure it works as expected.

Note: ‚ÄúLook-around‚Äù capabilities, including look-ahead and look-behind, are not supported.
